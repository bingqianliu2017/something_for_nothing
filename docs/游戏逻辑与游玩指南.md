# 游戏逻辑、功能与游玩指南

## 一、玩法概述

这是一款**躲避障碍**游戏：玩家控制蓝色方块，通过左右移动躲避从上方落下的红色障碍物，存活越久得分越高，碰到障碍物则游戏结束。

---

## 二、游戏逻辑（按流程）

### 1. 状态与流程

```
┌─────────────┐      点击「开始游戏」      ┌─────────────┐
│   Menu     │  ──────────────────────►  │    Game     │
│   主菜单   │                            │   游戏中    │
└─────────────┘                            └──────┬──────┘
       ▲                                               │
       │                                               │ 碰撞
       │                                               ▼
       │                                        ┌─────────────┐
       │         点击任意位置                    │   Result    │
       └────────────────────────────────────── │   结算      │
                                                └─────────────┘
```

### 2. 核心逻辑（GameScene）

每帧 `update()` 执行顺序:

```
┌──────────────────────────────────────────────────────────────┐
│ 1. 若 gameOver → 直接返回，不再更新                           │
└──────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────┐
│ 2. 输入 → 若正在触摸，取触摸点 pos，设置 player.targetX        │
│          (限制在屏幕左右边界内)                                │
└──────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────┐
│ 3. player.update(dt) → 向 targetX 平滑移动                    │
└──────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────┐
│ 4. spawnTimer 累加 → 每 1.2 秒在顶部随机 x 生成一个 Enemy      │
└──────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────┐
│ 5. 每个 Enemy.update(dt) → 向下移动 (vy ≈ 180~260)            │
└──────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────┐
│ 6. 过滤掉超出屏幕下方的 Enemy                                 │
└──────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────┐
│ 7. 得分 += floor(deltaTime * 10)，约每秒 +10 分                │
└──────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────┐
│ 8. CollisionSystem.check(player, enemies)                     │
│    若碰撞 → gameOver = true                                   │
│           → switchTo('Result', { score })                     │
└──────────────────────────────────────────────────────────────┘
```

## 三、当前已实现功能

| 模块 | 功能 | 说明 |
|------|------|------|
| **场景** | 主菜单 | 标题、开始按钮、进入动画 |
| | 游戏主场景 | 玩家、敌人生成、移动、碰撞、得分 |
| | 结算 | 显示得分、返回菜单 |
| **实体** | 玩家 | 50×50 蓝色方块，跟随触摸左右移动 |
| | 敌人 | 40×40 红色方块，从顶部随机 x 下落 |
| **系统** | 输入 | 触摸开始/移动/结束，点击分发 |
| | 碰撞 | AABB 检测玩家与敌人 |
| **资源** | ResourceManager | 图片/音效加载、缓存（当前场景未用到图片） |
| **工具** | Tween | 缓动插值（菜单/结算有简单手写动画） |
| **平台** | Canvas | tt.createCanvas()，按窗口尺寸初始化 |
| | 触摸 | tt.onTouchStart / Move / End |
| | 适配 | 窗口尺寸变化、竖屏 |

**未使用或未实现：**

- 图片资源（玩家/敌人为矩形绘制）
- 音效（ResourceManager 支持，未调用）
- Tween 库（菜单/结算为手写线性动画）
- 暂停、重新开始、难度变化等

## 四、如何游玩

### 操作方式

| 阶段 | 操作 |
|------|------|
| **主菜单** | 等标题和按钮动画完成 → 点击「开始游戏」 |
| **游戏中** | 按住并左右滑动控制蓝色方块，跟随触摸实时移动，躲避红色障碍物 |
| **结算** | 碰撞后自动进入 → 查看得分 → 点击任意位置返回菜单 |

### 操作说明示意

```
       屏幕
  ┌─────────────────────────┐
  │  得分: 1234             │  ← 顶部分数条
  ├─────────────────────────┤
  │                         │
  │      ▼ ▼ ▼ 红色下落     │
  │                         │
  │                         │
  │    ◄══════►             │  ← 手指左右滑动
  │      ■ 蓝块             │  ← 玩家跟随手指
  │                         │
  └─────────────────────────┘
```

### 3. 碰撞检测 (AABB)

```
   Player (蓝)
   ┌────────┐
   │        │      Enemy (红)
   │        │      ┌────┐
   │        │      │    │
   └────────┘      └────┘

   px1 < ex2  &&  px2 > ex1  &&  py1 < ey2  &&  py2 > ey1
   → 重叠 = 碰撞 → 游戏结束
```

### 4. 玩家移动逻辑

```
触摸点 x
    │
    ▼
┌───────┐
│ 触摸  │ → targetX = pos.x - player.width/2
└───────┘    限制在 [0, 屏幕宽 - 50]
    │
    ▼
┌───────┐    每帧: x += vx * dt
│ 玩家  │    vx = (targetX > x) ? 400 : -400
└───────┘    到 targetX 时 vx=0
```

### 5. 敌人生成与下落

```
时间线:
t=0s     t=1.2s   t=2.4s   t=3.6s  ...
  │         │         │         │
  ▼         ▼         ▼         ▼
生成E1   生成E2    生成E3    生成E4
  │         │         │         │
  ▼         ▼         ▼         ▼
x=随机    x=随机    x=随机    x=随机
(20 ~ 屏宽-20)

       y 轴
        ↑
    ────┼────  y=0 (顶部)
        │
        │   E1
        │     E2
        │       E3
        │         E4
        │
    ────┼────  y=屏高 (底部，超出则移除)
        ↓
```

### 6. 场景切换数据流

```
MenuScene.handleTap(点击按钮)
        │
        └──► switchTo('Game')
                    │
                    ▼
              GameScene.onEnter()
              - player, enemies 初始化
              - score=0, gameOver=false
                    │
                    │  (游戏进行中...)
                    │
    CollisionSystem.check() 返回 true
                    │
                    └──► switchTo('Result', { score: 1234 })
                                  │
                                  ▼
                            ResultScene.onEnter(data)
                            - this.score = data.score  // 1234
                            - 显示 "游戏结束"、"1234"、"点击返回"
                                  │
                                  │  handleTap(任意点击)
                                  │
                                  └──► switchTo('Menu')
```

### 2. 背景图 `bg_game.png`

**尺寸建议：**
- 竖屏：`750×1334` 或 `750×1624`（与常见机型分辨率一致，减少缩放）
- 横屏：`1334×750`
- 可略大（如 1080×1920），但体积需控制在 500KB 内

**渲染方式：**
- 使用 **Cover 截取填充**：保持比例，居中裁剪，填满画布，不拉伸
- 自动叠加**黑色遮罩**（约 45% 不透明度）降低亮度
- 可选**模糊**（`ctx.filter`，部分真机可能不支持）

**注意事项：**
- 主体内容尽量居中，边缘可能被裁切
- 避免过亮，游戏内会加遮罩
- 格式 PNG，支持透明（背景通常不透明）

### 3. 块图标（8 种）

**游戏内显示尺寸：** 40×40 px（会从 64 或 128 缩放）

**设计要点：**
- 正方形，内容居中，四周留少量边距
- 图标清晰、轮廓分明，缩小后仍可辨认
- 风格统一：西方奇幻、手绘感、色彩饱和

**类型对应（供生成提示词参考）：**
- 黄金 → 魔法金币、古代符文金币
- 钻石 → 蓝色水晶、魔法宝石
- 无敌 → 圣光护符、神圣徽记
- 护盾 → 骑士盾徽、防护纹章
- 扣血 → 荆棘刺球、暗红尖刺
- 即死 → 虚空之眼、骷髅符号
- 减速 → 寒冰碎片、霜冻结晶
- 幻想蘑菇 → 紫色发光蘑菇

### 4. 玩家 `player.png`

**游戏内显示尺寸：** 28×56 px（初始 28×28，随收集金币/钻石增大，最大 56×56）

**设计要点：**
- 正方形，角色居中

- 初始尺寸小于块（28 < 40），角色不宜过复杂

### 5. 通用注意事项

| 项目 | 说明 |
|------|------|
| **主包限制** | 抖音小游戏主包 4MB，总包 20MB，大图建议放 CDN 或分包 |
